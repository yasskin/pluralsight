Global and Local Scope

Last lesson we learned that when we write code in a browser it becomes a part of the global window object. Almost all the code we have written up until this point has been global. When something is global it means all code can access, change, and use it. This means we have to be very careful about creating anything that is global in scope so we don't accidentally do something we didn't intend.

When we create a variable inside of a function it can't be accessed from code outside of the function. Why? Because it's scope is local to that function, meaning it is invisible to any other code other than the other code inside of that function.
