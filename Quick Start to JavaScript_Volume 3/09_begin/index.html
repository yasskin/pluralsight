Coding Conventions

Up until this point we haven't been too strict when thinking about things like indentation or the amount of space between the code we type -- e.g. whether we type a function like var function () { ... } or var function(){ ... }. As it turns out, this and other small details are very important — especially when other people will be reading and working with your code.

One way to help write code in a standard way is to use a linting tool like JSLint. It is easy to use — it is built into Brackets already! It will give you a lot of errors. Try not to focus on fixing every error at first unless your code isn't working. For now try to focus on any inconsistency in formatting. The neater your code is the easier it becomes to read! While whitespace takes up more memory and makes your code slower, there are tools to minify, or remove formatting so code is as compact as possible, which you can use before it goes into production.
